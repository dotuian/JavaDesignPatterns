

状态模式和策略模式的区别：

状态模式与策略模式很容易混淆。有时候很难区分应当使用状态模式还是使用策略模式。

这个时候，一个简单的方式便是考试环境角色是否有明显的状态和状态的过渡。
如果环境角色只有一个状态，那么久应当使用策略模式。

策略模式的特点是：
一旦环境角色选择了一个具体的策略类，那么整个环境的生命周期里他都不会改变这个具体的策略类。

而状态模式则适用于另一种情况，即环境角色有明显的状态转移，在环境类的声明周期里面，会有几个不同的状态对象呗使用。


另一个微妙的区别在于：
策略模式的环境选择类自己选择一种具体的策略类；而状态模式的环境类是被外在的原因放进一个具体的状态中。

策略模式所选的策略往往不明显地告诉客户端它所选的具体策略；而状态模式则相反，
在状态模式里面，环境角色所处的状态是明显告诉给客户端的。
